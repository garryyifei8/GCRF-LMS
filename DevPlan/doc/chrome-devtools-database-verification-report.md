## 4. 数据一致性对比

### 4.1 数据来源分析

| 模块     | 数据来源 | 说明                      |
| -------- | -------- | ------------------------- |
| 登录认证 | MSW Mock | Token格式为mock-token-xxx |
| 图书列表 | MSW Mock | 返回100条模拟数据         |
| 仪表盘   | MSW Mock | 统计数据为固定值          |
| 读者列表 | MSW Mock | 前端页面未完成            |

### 4.2 数据差异对比

| 数据类型 | 数据库真实数据  | 前端显示数据       | 差异原因             |
| -------- | --------------- | ------------------ | -------------------- |
| 图书总数 | 50 条           | 100 条             | 前端使用MSW Mock数据 |
| 读者总数 | 40 人           | 5,000 人（仪表盘） | Mock统计数据         |
| 借阅记录 | 40 条（15活跃） | 未测试             | -                    |
| 馆藏量   | 实际按书统计    | 10,000册           | Mock固定值           |

### 4.3 ISBN对比验证

**数据库中存在的ISBN**:

- 9787115543998 (深入理解计算机系统)
- 9787111558422 (算法导论)
- 9787115428028 (Python编程)

**前端Mock数据中的ISBN**:

- 9787020002207 (三体)
- 9787020008735 (活着)
- 9787115428028 (Python编程) ✅ 匹配

**结论**: 部分ISBN在两个数据源中存在，但大部分数据不一致。

## 5. 问题发现与建议

### 5.1 发现的问题

| 问题编号 | 严重程度 | 描述                             | 影响范围     |
| -------- | -------- | -------------------------------- | ------------ |
| BUG-001  | 中       | 学生读者页面返回404              | 读者管理模块 |
| BUG-002  | 低       | Session丢失导致频繁重新登录      | 用户体验     |
| DATA-001 | 中       | 前端使用Mock数据，未连接真实后端 | 全模块       |
| DATA-002 | 低       | 仪表盘统计数据与数据库不一致     | 仪表盘       |

### 5.2 改进建议

1. **后端API集成**
   - 配置前端连接真实后端API而非MSW Mock
   - 在开发环境中可通过环境变量切换

2. **路由配置**
   - 检查并修复 `/readers/students` 路由配置
   - 确保菜单项与实际路由一致

3. **Session管理**
   - 实现Token持久化存储
   - 添加Token刷新机制

4. **数据同步**
   - Mock数据应与数据库Schema保持一致
   - 建议使用数据库种子数据生成Mock
