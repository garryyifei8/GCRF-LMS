## ⚠️ 关键问题诊断

### 问题现象
客户在 `http://192.168.2.121:3011` 访问系统时，登录显示"请求失败"错误。

### 根本原因（两个问题）

#### 问题 1: API 路径重复 ❌
```
配置组合:
  - Axios baseURL: /api
  - API URL: /api/v1/auth/login
  - 实际请求: /api + /api/v1/auth/login = /api/api/v1/auth/login

结果:
  - Nginx 返回: 405 Not Allowed
  - Mock Service Worker 无法拦截（路径不匹配）
```

#### 问题 2: Service Worker 不支持 ❌
```
访问方式: http://192.168.2.121:3011 (HTTP + IP地址)
浏览器限制: Service Worker 仅支持 HTTPS 或 localhost
结果: serviceWorkerSupported: false
影响: Mock API 完全无法工作
```

## ✅ v1.0.2 修复内容

### 修复 1: API 路径配置

**文件**: `.env.production`

**修改前 (v1.0.1)**:
```bash

```

**修改后 (v1.0.2)**:
```bash
# API基础路径 - 留空，因为 API 调用中已包含 /api 前缀
# Mock Service Worker 会拦截 /api/v1/... 路径

```

**效果**:
```
修复前: baseURL=/api + url=/api/v1/auth/login = /api/api/v1/auth/login ❌
修复后: baseURL=''  + url=/api/v1/auth/login = /api/v1/auth/login ✅
```

## 📝 技术细节

### API 路径构造逻辑

**Axios 请求构造**:
```javascript
// src/utils/request.js
const service = axios.create({
  baseURL: import.meta.env.VITE_API_BASE_URL || '/api'  // 从环境变量读取
})

// src/api/auth.js
export function login(data) {
  return request({
    url: '/api/v1/auth/login'  // API 路径已包含 /api
  })
}

// 最终请求 = baseURL + url
```

**v1.0.1 配置 (错误)**:
```
baseURL = '/api'
url = '/api/v1/auth/login'
实际请求 = '/api' + '/api/v1/auth/login' = '/api/api/v1/auth/login' ❌
```

**v1.0.2 配置 (正确)**:
```
baseURL = '' (空字符串)
url = '/api/v1/auth/login'
实际请求 = '' + '/api/v1/auth/login' = '/api/v1/auth/login' ✅
```

### MSW 拦截配置

**MSW Handler 路径**:

// src/mock/handlers/auth.js

**拦截逻辑**:

## ⚠️ 常见问题

### Q1: 为什么必须使用 HTTPS？
**A**: Service Worker 是浏览器安全功能，仅在以下环境可用：
- ✅ HTTPS 网站（https://192.168.2.121:3011）
- ✅ localhost（http://localhost:3011）
- ❌ HTTP + IP（http://192.168.2.121:3011）- 浏览器安全限制

### Q2: 可以只更新镜像不用 HTTPS 吗？
**A**: 不行。即使修复了 API 路径，HTTP + IP 访问仍然无法使用 Service Worker。两个问题必须同时解决。

### Q3: 自签名证书浏览器警告正常吗？
**A**: 正常。自签名证书未经 CA 认证，浏览器会显示警告。演示环境可以接受警告，生产环境建议使用正式证书。

### Q4: 如何在localhost测试（不用HTTPS）？
**A**: 在服务器本地部署标准 HTTP 版本即可：
```bash
docker run -d -p 3011:80 --name gcrf-web-admin gcrf-library-web-admin:amd64
# 在服务器上访问: http://localhost:3011
```

### Q5: 更新后旧版本的数据会丢失吗？
**A**: 不会影响。演示版使用 Mock 数据，每次刷新重置。真实数据需要连接后端数据库。

## 🎓 学习要点

### 为什么会出现路径重复？

这是配置层叠导致的经典问题：

1. **Axios 设计**: 支持 `baseURL` + `url` 拼接
2. **代码约定**: API 调用统一使用 `/api/v1/...` 路径
3. **环境配置**: 开发环境可能需要代理到后端 API 服务器
4. **Mock 场景**: Mock 数据在前端，不需要 baseURL 前缀

**最佳实践**:
- 生产环境 Mock 版本: `baseURL = ''` (空字符串)
- 开发环境后端联调: `baseURL = 'http://backend-api-server'`
