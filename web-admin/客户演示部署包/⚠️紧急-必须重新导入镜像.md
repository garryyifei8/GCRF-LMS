## ⚠️ 常见错误

### 错误 1: 没有删除旧镜像

**症状**: 导入后仍然是旧版本

**原因**: Docker 使用标签（tag），如果不删除旧镜像，新导入的可能不会覆盖

**解决**:
```bash
# 必须先删除
docker rmi gcrf-library-web-admin:amd64

# 然后重新导入
gunzip -c gcrf-web-admin-amd64-v1.0.0.tar.gz | docker load
```

### 错误 2: 导入了错误的文件

**症状**: 导入成功但问题依旧

**检查文件日期**:
```bash
ls -lh gcrf-web-admin-amd64-v1.0.0.tar.gz
```

应该显示:
```
-rw-r--r-- ... 25M Oct 21 13:15 gcrf-web-admin-amd64-v1.0.0.tar.gz
```

如果日期不是 `Oct 21 13:15` 或更晚，说明文件**不是最新的**！

### 错误 3: 容器使用缓存镜像

**症状**: 删除重建后仍然有问题

**彻底清理**:
```bash
# 停止所有相关容器
docker stop $(docker ps -a | grep gcrf | awk '{print $1}')

# 删除所有相关容器
docker rm $(docker ps -a | grep gcrf | awk '{print $1}')

# 删除所有相关镜像
docker rmi $(docker images | grep gcrf-library-web-admin | awk '{print $3}')

# 清理系统
docker system prune -f

# 重新导入
gunzip -c gcrf-web-admin-amd64-v1.0.0.tar.gz | docker load
```
