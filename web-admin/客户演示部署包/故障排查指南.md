# 故障排查指南

## 常见问题及解决方案

### 问题1: "Unable to find image 'gcrf-library-web-admin:latest' locally"

**错误信息**:
```
Unable to find image 'gcrf-library-web-admin:latest' locally
docker: Error response from daemon: Get "https://registry-1.docker.io/v2/": net/http: request canceled
```

**原因**: Docker镜像还没有被导入到本地，Docker尝试从远程仓库拉取但失败（网络问题或镜像不在公共仓库）。

**解决方案**:

#### 方案一: 手动导入镜像（推荐⭐）

```bash
# 1. 进入部署包目录
cd /path/to/客户演示部署包

# 2. 导入镜像
gunzip -c gcrf-web-admin-v1.0.0.tar.gz | docker load

# 3. 验证镜像
docker images | grep gcrf-library-web-admin

# 应该看到类似输出:
# gcrf-library-web-admin   latest   xxx   xxx   89.5MB

# 4. 启动容器
docker run -d \
  --name gcrf-web-admin \
  -p 3011:80 \
  --platform linux/amd64 \
  --restart unless-stopped \
  gcrf-library-web-admin:latest

# 5. 验证运行
docker ps | grep gcrf-web-admin
curl http://localhost:3011/health
```

#### 方案二: 使用导入脚本

```bash
cd /path/to/客户演示部署包
./部署脚本/import-image.sh
```

按照提示操作即可。

### 问题2: "The requested platform does not match"

**错误信息**:
```
The requested image's platform (linux/arm64) does not match the detected host platform (linux/amd64)
```

**原因**: 镜像在ARM64平台构建，但您的服务器是AMD64平台。

**解决方案**: 启动时添加 `--platform linux/amd64` 参数（脚本已自动处理）

```bash
docker run -d \
  --name gcrf-web-admin \
  -p 3011:80 \
  --platform linux/amd64 \
  --restart unless-stopped \
  gcrf-library-web-admin:latest
```

详细说明请查看 `跨平台部署说明.md`

### 问题3: 端口被占用

**错误信息**:
```
Error response from daemon: Ports are not available: exposing port TCP 0.0.0.0:3011
```

**解决方案**:

#### 选项A: 更换端口

```bash
docker run -d \
  --name gcrf-web-admin \
  -p 8080:80 \
  --platform linux/amd64 \
  --restart unless-stopped \
  gcrf-library-web-admin:latest

# 访问: http://localhost:8080
```

#### 选项B: 释放3011端口

```bash
# 查找占用端口的进程
lsof -i :3011

# 停止进程（根据上面的PID）
kill -9 <PID>
```

### 问题4: 容器启动失败

**检查步骤**:

```bash
# 1. 查看容器状态
docker ps -a | grep gcrf-web-admin

# 2. 查看容器日志
docker logs gcrf-web-admin

# 3. 查看详细信息
docker inspect gcrf-web-admin

# 4. 尝试交互式启动（调试用）
docker run -it --rm \
  -p 3011:80 \
  --platform linux/amd64 \
  gcrf-library-web-admin:latest \
  /bin/sh
```

**常见原因**:
- 端口冲突
- 权限问题
- Docker服务未正常运行
- 镜像损坏

### 问题5: 页面无法访问

**检查清单**:

```bash
# 1. 确认容器运行
docker ps | grep gcrf-web-admin
# 状态应该是 "Up"

# 2. 检查健康状态
curl http://localhost:3011/health
# 应该返回: healthy

# 3. 检查端口映射
docker port gcrf-web-admin
# 应该显示: 80/tcp -> 0.0.0.0:3011

# 4. 检查防火墙
# 确保3011端口未被防火墙阻止

# 5. 尝试其他地址
# 如果是远程服务器，使用服务器IP而不是localhost
curl http://<服务器IP>:3011/health
```

### 问题6: 页面显示空白

**解决方案**:

```bash
# 1. 清除浏览器缓存
# Chrome: Ctrl+Shift+Delete (Windows/Linux) 或 Cmd+Shift+Delete (Mac)
# Firefox: Ctrl+Shift+Delete (Windows/Linux) 或 Cmd+Shift+Delete (Mac)

# 2. 使用无痕模式
# Chrome: Ctrl+Shift+N (Windows/Linux) 或 Cmd+Shift+N (Mac)
# Firefox: Ctrl+Shift+P (Windows/Linux) 或 Cmd+Shift+P (Mac)

# 3. 检查浏览器控制台错误
# F12 打开开发者工具 -> Console标签

# 4. 验证Nginx配置
docker exec gcrf-web-admin nginx -t

# 5. 重启容器
docker restart gcrf-web-admin
```

**错误信息**:

```bash

# 验证

```

## 完整的手动部署流程

如果自动脚本失败，请按以下步骤手动部署：

```bash
# 1. 确认Docker运行
docker --version
docker info

# 2. 进入部署包目录
cd /path/to/客户演示部署包

# 3. 导入镜像
echo "导入镜像..."
gunzip -c gcrf-web-admin-v1.0.0.tar.gz | docker load

# 4. 验证镜像
echo "验证镜像..."
docker images | grep gcrf-library-web-admin

# 5. 清理旧容器（如果存在）
docker rm -f gcrf-web-admin 2>/dev/null || true

# 6. 启动容器
echo "启动容器..."
docker run -d \
  --name gcrf-web-admin \
  -p 3011:80 \
  --platform linux/amd64 \
  --restart unless-stopped \
  gcrf-library-web-admin:latest

# 7. 等待服务就绪
sleep 5

# 8. 验证运行
echo "验证运行..."
docker ps | grep gcrf-web-admin
curl http://localhost:3011/health

# 9. 访问系统
echo "✅ 部署完成!"
echo "访问地址: http://localhost:3011"
echo "管理员账号: admin / admin123"
```

## 网络问题处理

### 问题: 镜像导入超时或失败

**解决方案**:

```bash
# 1. 检查文件完整性
ls -lh gcrf-web-admin-v1.0.0.tar.gz
# 应该是 ~23MB

# 2. 测试解压
gunzip -t gcrf-web-admin-v1.0.0.tar.gz
# 没有输出表示文件完整

# 3. 分步导入
gunzip -c gcrf-web-admin-v1.0.0.tar.gz > gcrf-web-admin.tar
docker load -i gcrf-web-admin.tar
rm gcrf-web-admin.tar

# 4. 验证
docker images | grep gcrf-library-web-admin
```
