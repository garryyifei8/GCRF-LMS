# 跨平台部署说明 - 解决ARM64/AMD64架构问题

## ⚠️ 问题说明

**错误信息**:
```
The requested platform (linux/arm64) does not match the detected host platform (linux/amd64/v4)
and no specific platform was requested
```

**原因**: Docker镜像在ARM64架构(Apple Silicon Mac)上构建，但您的系统是AMD64架构(Intel/AMD处理器)。

---

## ✅ 解决方案(已更新)

**好消息**: 所有部署脚本已更新，自动支持跨平台！

### 方案一: 使用更新后的脚本(推荐⭐)

脚本已自动添加 `--platform linux/amd64` 参数，直接使用即可：

```bash
# 清理旧容器(如果有)
docker rm -f gcrf-web-admin 2>/dev/null || true

# 重新运行一键启动
./部署脚本/quick-start.sh
```

### 方案二: 手动指定平台

如果仍有问题，手动运行：

```bash
# 1. 导入镜像(如果还未导入)
gunzip -c gcrf-web-admin-latest-*.tar.gz | docker load

# 2. 启动容器，指定平台
docker run -d \
  --name gcrf-web-admin \
  -p 3011:80 \
  --platform linux/amd64 \
  --restart unless-stopped \
  gcrf-library-web-admin:latest

# 3. 验证运行
docker ps | grep gcrf-web-admin
```

### 方案三: 使用Docker Compose

Docker Compose配置也已更新：

```bash
# 确保旧容器已删除
docker-compose down

# 重新启动
docker-compose up -d

# 查看状态
docker-compose ps
```

## 📊 平台兼容性说明

### 支持的平台

| 平台 | 架构 | 状态 | 说明 |
|-----|------|------|------|
| Apple Silicon Mac | linux/arm64 | ✅ 原生支持 | M1/M2/M3芯片 |
| Intel Mac | linux/amd64 | ✅ 完全支持 | 使用platform参数 |
| Linux (Intel/AMD) | linux/amd64 | ✅ 完全支持 | 使用platform参数 |
| Windows (Intel/AMD) | linux/amd64 | ✅ 完全支持 | Docker Desktop + WSL2 |

### 性能说明

- **原生平台**(ARM64 on Apple Silicon): 最佳性能
- **模拟平台**(AMD64 on Apple Silicon, ARM64 on Intel): 略有性能损耗(通常<10%)
- 对于演示和测试环境，性能差异可忽略不计

## 🛠️ 故障排查

### 问题1: 仍然提示平台不匹配

**解决方案**:
```bash
# 1. 完全删除旧容器和镜像
docker rm -f gcrf-web-admin
docker rmi gcrf-library-web-admin:latest

# 2. 重新导入镜像
gunzip -c gcrf-web-admin-latest-*.tar.gz | docker load

# 3. 使用显式平台参数启动
docker run -d \
  --name gcrf-web-admin \
  -p 3011:80 \
  --platform linux/amd64 \
  gcrf-library-web-admin:latest
```

### 问题2: Docker不支持platform参数

**可能原因**: Docker版本太旧

**解决方案**:
```bash
# 检查Docker版本
docker --version
# 需要 Docker 20.10+

# 升级Docker
# macOS/Windows: 更新Docker Desktop
# Linux:
curl -fsSL https://get.docker.com | sh
```

### 问题3: 容器运行缓慢

**原因**: 跨平台模拟可能导致性能下降

**解决方案**:
1. **临时方案**: 继续使用，性能损失通常<10%
2. **最佳方案**: 在相同架构的机器上重新构建镜像

```bash
# 在AMD64机器上重新构建
cd web-admin
docker build --platform linux/amd64 -t gcrf-library-web-admin:latest .
```

## 🔄 为不同平台重新构建镜像

### 方法一: 单平台构建

**在AMD64平台(Intel/AMD)上**:
```bash
cd web-admin
docker build --platform linux/amd64 -t gcrf-library-web-admin:latest .
```

**在ARM64平台(Apple Silicon)上**:
```bash
cd web-admin
docker build --platform linux/arm64 -t gcrf-library-web-admin:latest .
```

### 方法二: 多平台构建(推荐)

使用Docker Buildx创建支持多平台的镜像：

```bash
# 1. 创建builder实例
docker buildx create --name multiplatform --use

# 2. 构建多平台镜像
docker buildx build \
  --platform linux/amd64,linux/arm64 \
  -t gcrf-library-web-admin:latest \
  --load \
  .

# 3. 验证
docker buildx imagetools inspect gcrf-library-web-admin:latest
```

### 方法三: 分别导出不同平台镜像

```bash
# 构建AMD64版本
docker build --platform linux/amd64 -t gcrf-library-web-admin:amd64 .
docker save gcrf-library-web-admin:amd64 | gzip > gcrf-web-admin-amd64.tar.gz

# 构建ARM64版本
docker build --platform linux/arm64 -t gcrf-library-web-admin:arm64 .
docker save gcrf-library-web-admin:arm64 | gzip > gcrf-web-admin-arm64.tar.gz
```
