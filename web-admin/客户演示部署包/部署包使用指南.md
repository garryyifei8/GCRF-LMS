## 📦 快速部署（3步）

### 步骤 1: 导入镜像
```bash
cd /path/to/客户演示部署包
gunzip -c gcrf-web-admin-amd64-v1.0.0.tar.gz | docker load
```

### 步骤 2: 启动容器
```bash
docker run -d \
  --name gcrf-web-admin \
  -p 3011:80 \
  --restart unless-stopped \
  gcrf-library-web-admin:amd64
```

### 步骤 3: 访问系统
浏览器打开: http://localhost:3011

**登录账号**: `admin` / `admin123`

## 🌐 首次访问说明

### 正常流程
1. **首次访问** - 显示"初始化中..."（1-2秒）
2. **自动刷新** - Service Worker 激活
3. **显示登录页** - 可以正常登录使用

### 为什么需要刷新？
Mock Service Worker 使用浏览器的 Service Worker 技术拦截 API 请求。首次访问时：
- Service Worker 注册但未激活
- 需要刷新页面让 Service Worker 接管控制
- 这是一次性操作，后续访问无需再刷新

### 浏览器兼容性
- ✅ Chrome 90+
- ✅ Firefox 88+
- ✅ Safari 14+
- ✅ Edge 90+

## ❓ 常见问题速查

### Q1: 导入镜像时提示"file not found"
**原因**: 当前目录不正确
**解决**:
```bash
cd /path/to/客户演示部署包
ls gcrf-web-admin-amd64-v1.0.0.tar.gz  # 确认文件存在
```

### Q2: 启动容器时提示"image not found"
**原因**: 忘记导入镜像
**解决**: 先执行步骤1导入镜像，或使用 `./部署脚本/0-import-only.sh`

### Q3: 访问页面一直显示"初始化中..."
**原因**: Service Worker 激活时间过长
**解决**:
1. 等待5秒
2. 手动刷新页面（F5）
3. 清除缓存后重试

### Q4: 登录显示"网络连接失败"
**原因**: Service Worker 未正确激活
**解决**:
1. 刷新页面（F5）
2. 打开浏览器控制台（F12）查看 [MSW] 日志
3. 访问 http://localhost:3011/debug.html 诊断
4. 使用无痕模式测试

### Q5: Mac M1/M2 提示平台不匹配
**原因**: 本镜像为 AMD64 架构
**解决**: Mac M1/M2 可以通过 Rosetta 2 运行，性能略有影响但完全可用
