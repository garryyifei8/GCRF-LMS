# 国创睿峰智能图书馆管理系统 - 快速演示指南

## 🚀 快速启动

### 1. 启动开发服务器

```bash
cd web-admin
npm run dev
```

浏览器自动打开 `http://localhost:3011`

### 2. 登录系统

使用以下任一测试账号登录:

| 账号 | 密码 | 角色 | 权限 |
|------|------|------|------|
| **admin** | **admin123** | 系统管理员 | 全部权限 |
| librarian | lib123 | 图书管理员 | 流通、图书、读者查看 |
| operator | op123 | 操作员 | 仅借还书 |

**推荐使用 admin 账号进行完整演示**

---

### 场景2: 图书借出流程 ⭐⭐⭐⭐⭐

**路径**: 流通管理 > 图书借出

**演示步骤**:

1. **扫描/输入读者证号**
   - 输入: `RD00000001` - `RD00000200` 任意读者
   - 系统自动显示读者信息和借阅状态

2. **扫描/输入图书条码**
   - 输入: `BK00000001` - `BK00000100` 任意图书
   - 系统检查借阅资格(超限、超期)

3. **确认借出**
   - 自动记录借阅时间、应还时间
   - 更新图书状态

**亮点**:
- 自动校验借阅资格
- 实时库存检查
- 支持批量借阅

---

### 场景3: 图书归还流程 ⭐⭐⭐⭐

**路径**: 流通管理 > 图书归还

**亮点**:

## ❓ 常见问题应对

### Q: 数据是真实的吗?

A: 当前使用Mock数据进行演示,数据结构和交互逻辑与真实系统一致。实际部署后将连接真实数据库。

### Q: AI功能能实际使用吗?

A: AI功能的UI已完成,实际的推荐算法、NLP引擎等需要后端AI服务支持,目前使用模拟数据演示交互流程。

### Q: 移动端支持吗?

A: 当前为Web管理端,移动端(微信小程序)正在开发中,将提供读者自助借阅、预约、推荐等功能。

### Q: 人脸识别功能可用吗?

A: 人脸识别组件已开发,实际使用需要集成人脸识别硬件设备和SDK。

### Q: 支持哪些浏览器?

A: 支持Chrome、Firefox、Edge、Safari最新两个版本,推荐使用Chrome浏览器获得最佳体验。

### Q: 数据会保存吗?

A: Mock演示环境下,数据存储在内存中,刷新页面会重置。实际系统会持久化存储到数据库。
