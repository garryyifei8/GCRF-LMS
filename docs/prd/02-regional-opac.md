## 1. 角色与权限

| 角色         | 权限                                            |
| ------------ | ----------------------------------------------- |
| 游客 (GUEST) | 检索 / 浏览 / 看详情 / 看排行榜（**不可预约**） |

## 2. 功能模块（来自需求 1-6）

### 2.1 跨校馆藏检索

**功能**：

- 单条件检索（题名、作者、ISBN、出版社）
- 高级检索（多条件 AND/OR + 字段限定）
- 按学校筛选 / 全区聚合
- 实时显示馆藏流通状态（在馆 / 借出 / 预约中）
- 检索结果图文 / 文字两种展示

**UI**：

- 顶部 Google 风格搜索栏，支持智能联想
- 左侧筛选面板（学校 / 中图分类 / 出版年 / 状态）
- 右侧结果列表：图文卡片 + 列表两种切换

**关键 API**：

- `GET /api/v1/opac/search?q=&clc=&school=&adv=`
- `GET /api/v1/opac/suggest?q=`（自动补全）
- `GET /api/v1/opac/books/{isbn}`（含跨校在馆状态）

### 2.2 中图法浏览

**功能**：

- 按中国图书馆分类法（CLC）22 小类树形浏览
- 新书按分类浏览
- 点击分类 → 展开下属 → 显示该分类馆藏

**UI**：

- 左树右内容布局
- 分类节点显示馆藏册数

**关键 API**：

- `GET /api/v1/opac/clc/tree`（中图法分类树）
- `GET /api/v1/opac/clc/{code}/books?school=&page=`

### 2.3 排行榜

**功能**：

- 借阅排行榜（本周/本月/本学期，按数量）
- 检索排行榜（最热搜的书）
- 查看排行榜（最热看详情的书）
- 检索词排行榜（热搜词）
- 区域 vs 学校 两个维度

**UI**：

- 卡片式榜单，每条带封面 + 借阅次数 + 排名变化箭头
- 切换时间窗口（周/月/学期）

**关键 API**：

- `GET /api/v1/opac/rankings/borrow?range=week|month|term&school=`
- `GET /api/v1/opac/rankings/search?range=`
- `GET /api/v1/opac/rankings/view?range=`
- `GET /api/v1/opac/rankings/keywords?range=`

### 2.4 智能推荐

**功能**：

- 基于查看图书的相关推荐（数据挖掘）
- 协同过滤：相似读者借阅
- 内容相似：同分类同主题
- 推荐分数 + 推荐理由

**UI**：

- 详情页底部"相关推荐"
- 列表 5-10 本，带评分 + 简短理由

**关键 API**：

- `GET /api/v1/opac/recommend/related?isbn=&limit=10`

### 2.5 OPAC 实时信息

**功能**：

- 馆藏在馆 / 借出 / 预约 状态实时（每 30 秒缓存）
- 当前热门借阅情况（首页 widget）
- 新书上架（最近 30 天）

**关键 API**：

- `GET /api/v1/opac/new-arrivals?school=&days=30`
- `GET /api/v1/opac/availability/{isbn}/by-school`

## 4. 非功能要求

- QPS 上限：单区域峰值 1000 QPS（开学日检索量）
- P95 延迟：检索 < 500ms / 详情 < 200ms
- 缓存：Redis 30s TTL；ES 跨校索引每 5min refresh
- 反爬：Cloudflare WAF + IP 限流（10 req/s/IP）
- SEO：服务端渲染（SSR），便于学校官网友链

## 5. 技术栈

- 前端：Vue 3 + Vite SSR / 或 Nuxt 3
- 后端：`opac-service` (8092) + ES 8 + Redis
- 检索：ES 跨校 alias 聚合查询

## 6. 验收标准

- [ ] 跨校检索能在 500ms 内返回 1000 条结果
- [ ] 中图法 22 小类浏览树正确
- [ ] 4 类排行榜（借阅/检索/查看/检索词）实时刷新
- [ ] 推荐相关图书命中率 > 70%（人工抽样）
- [ ] 反爬虫拦截恶意请求 > 95%
