## 1. 产品愿景

为教育局打造一体化的区域图书馆智慧云平台，覆盖**1 个云中心 + 4 类前端**，
让教育局到学校到家长在同一身份、同一馆藏、同一阅读分析框架下协作。

```
教育局 ─┐                 ┌── 学校管理员（Web + Edge Agent）
       │                 ├── 学生（OPAC + 微信）
       └── 区域云平台 ────┤
                        ├── 教师（多平台）
                        └── 家长（微信）
```

## 3. 价值主张矩阵

| 角色     | 痛点                                           | 我们的解                                                   |
| -------- | ---------------------------------------------- | ---------------------------------------------------------- |
| 教育局   | 200 校馆藏数据散乱、达标率难统计、不适宜书难管 | 区域云平台一屏看全区、馆藏标准自动测算、不适宜书库定期扫描 |
| 学校馆员 | 编目慢、采购无依据、断网就停业                 | CNMARC 自动获取、智能采购推荐、Edge Agent 离线借还         |
| 教师     | 给班级布置阅读、看不到效果                     | 班级阅读报表、智能推荐书单、阅读测评                       |
| 学生     | 不知道图书馆有什么书、借完忘还                 | OPAC 跨校检索、微信借阅查询、到期推送                      |
| 家长     | 不知道孩子在读什么                             | 微信家长端查阅读报告                                       |

## 4. 关键非功能需求

| 维度   | 要求                                                             |
| ------ | ---------------------------------------------------------------- |
| 可用性 | 学校借还核心功能 4 个 9（断网由 Edge Agent 兜底）；区域云 3 个 9 |
| 性能   | OPAC 检索 P95 < 500ms；借还操作 P95 < 200ms（含本地）            |
| 规模   | 单区域 ≤ 200 学校 / ≤ 50 万读者 / ≤ 400 万册图书                 |
| 安全   | JWT + RBAC + 数据范围；学校间逻辑隔离；WSS mTLS                  |
| 合规   | 数据 100% 落地教育局机房；不适宜书库每日扫描；可审计             |
| 移动   | 微信公众号 H5 + 小程序；支持 iOS / Android 主流机型              |

## 5. 边界与非目标

**做**：

- 4 平台软件 + 区域中心部署 + Edge Agent
- 与一卡通 / 钉钉 / 微信开放平台对接
- 馆藏标准 + 不适宜书库 + 智能分析

**不做**：

- 不替代物理硬件（条码枪、防盗仪、自助借还机）— 仅提供软件接口
- 不实现互联网级 OPAC（区域内访问足够）
- 不替代教育局既有 BI（通过数据密钥嵌入）

## 6. 上线路径

```
M1 (4-6 周)  区域云后台 + IAM SSO + OPAC 基础检索 → 教育局演示
M2 (6-8 周)  馆藏标准 + 不适宜书库 + Edge Agent + 微信 H5 → 1 校投产
M3 (6-8 周)  小程序 + 数据中心 + 阅读测评 → 多校铺开
```

详细里程碑见主 spec Appendix C。
