## 1. 信息架构概述

### 1.1 设计原则

| 原则 | 说明 | 应用实践 |
|------|------|----------|
| **清晰性** | 信息组织逻辑清晰、易于理解 | 采用任务导向的分组方式 |
| **一致性** | 跨平台术语、概念、结构保持一致 | 统一核心业务实体命名 |
| **可扩展性** | 架构能够适应未来功能扩展 | 预留AI服务、增值服务模块 |
| **可发现性** | 用户能够快速找到所需功能 | 不超过3级导航深度 |
| **场景化** | 针对不同用户角色定制信息结构 | 管理端、读者端差异化设计 |


### 2.3 信息深度控制

**三级导航原则**: 任何功能不超过3次点击到达

| 导航层级 | 深度 | 示例 | 说明 |
|---------|------|------|------|
| **L1 一级导航** | 全局菜单 | 流通管理 | 顶部/侧边主导航 |
| **L2 二级导航** | 功能分组 | 图书借出 | 同类功能聚合 |
| **L3 内容页面** | 具体内容 | 借出操作页 | 最终操作页面 |

**面包屑导航**: 首页 > 流通管理 > 图书借出

## 4. 导航结构设计

### 4.1 Web管理端导航

#### 4.1.1 侧边栏导航 (推荐)

**布局**: 左侧固定侧边栏 + 顶部面包屑

```
┌─────────┬──────────────────────────────────┐
│         │ 面包屑: 首页 > 流通管理 > 图书借出│
│ Logo    ├──────────────────────────────────┤
│         │                                  │
├─────────┤                                  │
│ 首页    │                                  │
│         │        主内容区                  │
├─────────┤                                  │
│ 流通管理│                                  │
│  借出   │                                  │
│  归还   │                                  │
│  记录   │                                  │
├─────────┤                                  │
│ 图书管理│                                  │
│  编目   │                                  │
│  典藏   │                                  │
│  查询   │                                  │
│  盘点   │                                  │
└─────────┴──────────────────────────────────┘
```

**优点**:
- 一级、二级菜单同时可见，导航清晰
- 适合功能模块较多的管理系统
- 菜单项可展开/收起，节省空间

#### 4.1.2 顶部导航 (备选)

**布局**: 顶部Tab导航 + 二级下拉菜单

```
┌───────────────────────────────────────────────┐
│ Logo  首页 | 流通 | 图书 | 读者 | 统计 | 系统  │
├───────────────────────────────────────────────┤
│                                               │
│                                               │
│               主内容区                         │
│                                               │
│                                               │
└───────────────────────────────────────────────┘
```

**优点**:
- 页面宽度利用率高
- 适合功能模块不太多的系统
- 视觉更简洁

### 4.2 小程序端导航

#### 4.2.1 底部Tab导航

**固定4个Tab**: 首页、馆藏、借阅、我的

**优势**:
- 符合移动端用户习惯
- 核心功能一触即达
- Tab之间平行切换

#### 4.2.2 页面内导航

**顶部胶囊按钮**: 返回、首页
**页面内Tab**: 子功能切换 (如借阅页的"当前借阅"、"借阅历史")
**浮动按钮**: 智能助手、搜索、扫码

### 4.3 面包屑导航规范

**Web管理端**:
```
首页 > 一级菜单 > 二级菜单 > 当前页面
示例: 首页 > 流通管理 > 图书借出
```

**小程序端**:
```
顶部导航栏标题 + 返回按钮
无需面包屑 (移动端空间有限)
```

## 5. 信息层级关系

### 5.1 信息优先级划分

| 优先级 | 信息类型 | 展示方式 | 示例 |
|-------|---------|---------|------|
| **P0 核心信息** | 用户当前最需要的信息 | 大标题、突出显示、页面顶部 | 图书详情页的书名、作者 |
| **P1 关键信息** | 支撑核心任务的信息 | 次级标题、清晰可见 | 馆藏状态、借阅规则 |
| **P2 辅助信息** | 补充说明、背景信息 | 正文、可展开区域 | 图书简介、目录 |
| **P3 扩展信息** | 可选查看的额外信息 | 折叠区、标签页、弹窗 | 评价、书评 |

### 5.2 信息分组策略

**按任务分组**: 流通管理、图书管理、读者管理
**按频率分组**: 常用功能、偶尔使用、高级功能
**按角色分组**: 管理员功能、馆长功能、操作员功能

### 5.3 信息呈现密度

| 设备类型 | 信息密度 | 原则 |
|---------|---------|------|
| **PC端管理** | 高密度 | 单屏尽量多展示,减少滚动,提升效率 |
| **移动端小程序** | 低密度 | 单屏聚焦核心任务,留白充足,易于点击 |

## 6. 跨平台信息一致性

### 6.1 核心术语统一

| 功能 | 统一术语 | 禁止使用 |
|------|---------|----------|
| 借阅图书 | "借阅" / "借书" | "租借"、"取书" |
| 归还图书 | "归还" / "还书" | "退还"、"退书" |
| 图书预约 | "预约" | "订阅"、"申请" |
| 读者证 | "读者证" | "借书证"、"会员卡" |
| 馆藏地 | "馆藏位置" / "书架号" | "存放地"、"地点" |

### 6.2 状态标识统一

| 状态 | 颜色 | 图标 | 文字 |
|------|------|------|------|
| **在架** | 绿色 #52C41A | ✓ | "在架" |
| **借出** | 灰色 #999 | - | "已借出" |
| **预约** | 橙色 #FA8C16 | ⏱ | "预约中" |
| **逾期** | 红色 #F5222D | ⚠ | "逾期X天" |
| **剔旧** | 灰色 #CCC | × | "已剔旧" |

### 6.3 数据展示一致性

**时间格式**:
- 日期: 2025-10-10
- 时间: 14:30:00
- 日期时间: 2025-10-10 14:30

**数量格式**:
- 整数: 1,234
- 小数: 12.34

**借阅规则表述**:
- "最多可借X本,借期X天"

### 6.4 操作流程一致性

**借阅流程**:
1. 验证读者身份
2. 扫描图书条码
3. 确认借阅
4. 提示成功

**跨平台保持一致** (Web管理端、小程序、自助机)

## 7. 信息架构优化建议

### 7.1 可发现性优化

- **全局搜索**: Web端顶部、小程序首页提供全局搜索
- **智能助手**: 任何页面都能快速呼出AI助手

- Web端: 预留"增值服务"一级菜单

- AI服务作为独立模块,可按需启用

### 7.3 无障碍设计

- 清晰的信息层级
- 充足的点击目标尺寸 (小程序最小44px)
- 高对比度的文字和背景
- 支持字体大小调整
